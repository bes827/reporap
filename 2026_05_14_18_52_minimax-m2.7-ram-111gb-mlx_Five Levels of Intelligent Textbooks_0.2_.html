Five Levels of Intelligent Textbooks
  • Source: Dan McCreary, Medium.com (2024-11-19)
  • Topic: A Proposed Scale for AI-Powered Textbooks
  • Author(s): Dan McCreary
Overview of Intelligent Textbook Levels
  • 5 levels exist from static to fully autonomous AI
  • Higher levels require more investment but offer proportional rewards
  • Generative AI is ↓ costs of generating smart content
  • Model inspired by SAE autonomous driving levels
Level 1 — Static Textbooks
  • Traditional printed or digital textbooks
  • No interactive elements
  • Content: text, static images, figures, tables, charts
  • No dynamic content generation
  • No internal/external links to resources like MicroSims
  • Best for: simple, short, straightforward content delivery
  • Over 90% of college textbooks today are Level 1
Level 2 — Interactive Content Textbooks
  • Essential interactive elements added
  • Keyword search functionality
  • Clickable links to internal content
  • External resource links (e.g., Wikipedia)
  • Embedded videos
  • Simple quizzes embedded in text
  • Users engage with multimedia directly
2.1. Tools Used for Level 2
  • Uses mkdocs-material (free, open-source)
  • Built with Visual Studio Code
  • Easy to configure and maintain
2.2. Features Enabled by Level 2
  • Search function: single config file element
  • Deep links to each page section
  • Social media preview cards (Open Graph Protocol)
  • Google Analytics for usage monitoring
  • Feedback forms on every page footer
  • Glossary linking with auto-completion in markdown
2.3. MicroSims in Level 2
  • Interactive simulations running in browser
  • Generated by AI tools like ChatGPT o1-preview
  • One of easiest ways to add interactivity
  • Cost-effective for any interactive textbook
2.4. Self-Assessment Quizzes in Level 2
  • Questions embedded directly in text
  • "Show Answer" button reveals responses
  • Students click to reveal answers
2.5. Limitations of Level 2
  • Content does not change based on student background
  • One-size-fits-all solution
  • Everyone gets same content regardless of prior knowledge
  • No personalization based on individual needs
Level 2.9 — Getting Ready for Level 3
  • Transition stage between Level 2 and Level 3
  • Requires integration with student needs
  • Free public hosting may not be sufficient
  • Process is difficult and time-consuming
  • Requires continual feedback and testing
Level 3 — Adaptive Textbooks
  • Content adjusts based on user input/performance
  • Adapts difficulty levels
  • Provides additional resources when needed
  • Uses deterministic rules for personalization
3.1. How Level 3 Works
  • Concept graph traversal algorithms personalize paths
  • Selects from fixed levels of curated content
  • Based on simple assessment scores
  • Delay-time-based rules for content delivery
  • Can skip content if users show high proficiency
  • Review quiz failure --> content reviewed before new material
3.2. Personal Learning Graphs
  • Records concepts each student has mastered
  • Stored on learning graph for each student
  • Creates personalized learning path
  • Each student needs their own unique graph
3.3. Limitations of Level 3
  • May lack real-time responsiveness
  • No LLM-generated custom lesson plans
  • No dynamic MicroSim generation at runtime
  • Still uses pre-designed content
Privacy and Legal Concerns (Level 3+)
  • Personal data requires careful protection
  • Must comply with GDPR in EU
  • Must comply with FERPA in US
  • Regulations impact web cookies stored in browsers
  • Regulatory compliance ↑ hosting costs significantly
  • Outside auditors must certify compliance
  • Creates opportunity for startups providing infrastructure
Level 4 — Textbooks with Chatbots
  • Combines concept graph algorithms with LLMs
  • Builds chatbot to answer student questions
  • Uses GraphRAG architecture for intelligent responses
4.1. GraphRAG Components
  • NLP pipelines
  • LLMs
  • Embeddings
  • Vector stores
  • Converts documents into graphs
4.2. How Level 4 Functions
  • Learning Graph becomes "ground truth"
  • Intelligent agents ride on curated graphs
  • Real-time feedback from chatbot
  • Adjusts content recommendations dynamically
  • Responsive question-answering systems
4.3. Features of Level 4
  • Knows user context
  • Provides tailored responses
  • Recommendations based on student progress
  • Protects sensitive data
  • Minimizes cost of using large LLMs
  • Smaller LLMs (e.g., Ollama on local GPUs) may suffice
4.4. Timeline and Adoption
  • Expected in learning institutions next year (2025)
  • Forward-thinking thought leaders first
  • Commercial organizations like Khan Academy already working on these features
4.5. Limitations of Level 4
  • Still has limitations
  • Teachers and instructors will remain necessary
  • Not fully autonomous learning
Level 5 — Autonomous AI Textbooks
  • Fully AI-driven adaptive intelligent textbooks
  • Imagined in Neal Stephenson's The Diamond Age (1995)
  • Cyberpunk novel: AI tablet generating animated stories in real-time
5.1. Vision of Level 5
  • Deep understanding of individual student's knowledge
  • Knows each student's learning goals
  • Answers complex questions through NLP
  • Generates customized lessons in real-time
5.2. Current Limitations
  • Like self-driving cars in heavy snowstorm at night
  • Requires specialized hardware (graph-traversal chips)
  • Current CPU hardware not sufficient
  • LLMs still unreliable for full autonomy
  • Not happening in near term
5.3. Future Expectations
  • Grandchildren will have comprehensive teaching assistants
  • Mimics one-on-one tutor experience
  • Goes beyond single textbook
  • Always supports learning objectives
  • Protects learning graph privacy
  • Fully autonomous and interactive learning agents
Breaking Through the Layers
  • 5 levels are just a model to help evaluate progress
  • Model is easy to change and customize
  • Can skip layers if needed
  • Example: Level 2 --> directly to Level 4 features using LLMs
  • Skipping avoids regulatory compliance burdens
  • Prompts may contain student data but never stored on website
Conclusion and Future Outlook
  • Scale provides framework for evaluating textbooks
  • Guides educators, developers, publishers in development journey
  • Many unknowns remain in the field
  • LLM quality will ↑ (confident prediction)
  • Standards for exchanging textbooks between systems not yet available
  • Depends on standard graph databases to execute agents
6.1. Student Preferences
  • Students prefer hands-on interaction over static textbooks
  • Demand for interactive content growing
6.2. Author's Hope
  • All children on Earth should have equal learning opportunities
  • Regardless of country, race, gender, socioeconomic background
  • Intelligent textbooks can help bridge gaps
Key Terms and Definitions
Term Definition
Learning Graph Concept dependency graphs for customized learning paths
MicroSims Interactive simulations running in browser
GraphRAG Graph retrieval-augmented generation architecture
LLM Large Language Model (AI system)
LMS Learning Management System
GDPR EU data protection regulation
FERPA US student privacy law
Open Graph Protocol Social media metadata standard for link previews
