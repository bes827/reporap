Model Variants & Performance
  • - Deepseek 3.2 is released in two main versions:
  • - Deepseek 3.2: The standard 'thinking' model, optimized for token efficiency.
  • - Deepseek 3.2 Special: A high-compute 'max thinking' model, prioritizing performance over token efficiency.

Core Technological Innovations
  • - Deepseek 3.2 introduces several novel algorithmic breakthroughs to enhance efficiency and capability.
1. Deepseek Sparse Attention (DSA)
  • - An updated, highly efficient attention mechanism designed for long context scenarios.
Computational Complexity Reduction
  • - Reduces core attention complexity from quadratic O(L²) to a more linear O(L * K).
  • - This translates from 'exploding costs' to 'controlled costs' as context length (L) increases.

Primary Benefit
  • - Allows for a significant increase in the model's context window without the typical quadratic increase in computational cost and speed sacrifice.


2. Scalable Reinforcement Learning (RL) Framework
  • - A robust RL protocol was implemented with a significant compute allocation for post-training.
Compute Investment
  • - The post-training computational budget exceeded 10% of the pre-training cost, a substantial investment compared to previous models.

Performance Outcome
  • - Unlocks advanced capabilities, allowing the high-compute variant (3.2 Special) to surpass GPT-5 and match Gemini 3.0 Pro in reasoning proficiency.


3. Large-Scale Agentic Task Synthesis Pipeline
  • - A novel pipeline was developed to systematically generate synthetic training data at scale, specifically for agentic tasks and tool use.
Data Generation Scale
  • - Generated data from over 1,800 distinct environments.
  • - Created 85,000 complex prompts for agentic tasks.

Objective & Outcome
  • - Aims to integrate reasoning into tool-use scenarios.
  • - Significantly enhances the model's generalization and instruction-following capabilities in agentic contexts.



Model Architecture & Specifications
  • - The model is a large yet efficient Mixture of Experts (MoE) model.
Parameters
  • - Total Parameters: 671 billion
  • - Active Parameters (at inference): 37 billion

Hardware Requirements (VRAM)
  • - For FP8 precision: 700 GB of VRAM.
  • - For full BF16 precision: 1.3 TB of VRAM.


Tool Use Capability
  • - The model is especially good at tool calling, significantly narrowing the performance gap between open-source and closed-source LLMs.
  • - While very strong, it is noted to be slightly below the absolute frontier models in this specific area.
