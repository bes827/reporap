1. Command-Line Habit Tracker
  • Offline, minimal CLI tool; Tracks streaks for habits; Goal: Keeps user honest w/o fancy UI/APIs.
Core Mechanism
  • Uses JSON for data persistence; Stores habit logs locally; Calculates streaks based on daily entries.
Data Storage
  • FILE = 'habits.json'; `json.load()` → retrieve data; `json.dump()` → save data w/ `indent=2` for readability.
Logging Habit
  • `log(habit)` function; Records `datetime.date.today().isoformat()` for habit; `data.setdefault(habit, []).append(today)` → adds date to habit list.
Calculating Streak
  • `streak(habit)` function; Sorts dates for a habit; Iterates `reversed(dates)` from `today`; `(today - datetime.date.fromisoformat(d)).days == streak` → checks for consecutive days.
Key Learnings
  • Date logic (`datetime` module); JSON persistence; CLI application development.
Special Aspect
  • Practical daily use; Builds CLI habits; No external dependencies/APIs.
2. Local Text Summarizer
  • Summarizes long text locally; Learns user's writing tone; Avoids external APIs.
Core Mechanism
  • Uses TF-IDF for feature extraction; Cosine similarity to find most similar document; Summarizes by returning a snippet of the top similar document.
Libraries Used
  • `sklearn.feature_extraction.text.TfidfVectorizer`; `sklearn.metrics.pairwise.cosine_similarity`.
Summarization Process
  • `summarize(text, docs)` function; `TfidfVectorizer(stop_words='english')` → converts text to TF-IDF features; `cosine_similarity(X[-1], X[:-1])` → compares input text to training docs; `top_doc = docs[sim.argmax()]` → selects most similar document; Returns first 200 chars of top doc + '...'.
Key Learnings
  • Natural Language Processing (NLP) basics; TF-IDF concept; Cosine similarity for text comparison.
Special Aspect
  • Personalized summarization; Learns user's specific writing style; No reliance on large language models (LLMs) like GPT.
3. Screenshot Diff Tool
  • Compares two screenshots pixel-by-pixel; Highlights visual differences; Utility: Quick visual QA for devs.
Core Mechanism
  • Uses `PIL` (Pillow) library; Opens two images; Calculates pixel-level difference; Saves difference as a new image.
Libraries Used
  • `PIL` (Pillow) library: `Image`, `ImageChops`.
Difference Calculation
  • `diff(img1, img2, out)` function; `Image.open(img1), Image.open(img2)` → loads images; `ImageChops.difference(i1, i2)` → computes pixel-wise difference; `diff.save(out)` → saves the resulting difference image.
Key Learnings
  • Image processing fundamentals; Pixel manipulation; Practical application in visual quality assurance.
Special Aspect
  • Quick visual QA for developers; 15-line script for immediate utility; Verifies UI changes post-deployment.
4. 'Hot Folder' Automation Butler
  • Directory watcher; Automatically processes incoming files; Acts as a mini-Zapier for local files.
Core Mechanism
  • Continuously monitors a specified folder (`inbox`); Identifies file types (e.g., `.jpg`, `.txt`); Applies specific processing based on file type; Moves processed files to another folder (`processed`).
Libraries Used
  • `time`, `os`, `shutil`, `PIL.Image`.
Automation Loop
  • `while True:` loop for continuous monitoring; `time.sleep(3)` → pauses for 3 seconds between checks; `os.listdir(FOLDER)` → lists files in `inbox`.
File Processing Examples
  • `.jpg` files: `Image.open()`, `img.save(..., optimize=True, quality=70)` → optimizes image quality to 70%; `.txt` files: `shutil.move()` → moves file directly to `processed` folder.
Key Learnings
  • File system watching; Automation loops; File manipulation (`os`, `shutil`); Image optimization (`PIL`).
Special Aspect
  • Understanding how Dropbox-like tools function; Practical local file automation; Reduces manual file management.
5. Markdown Dashboard Generator
  • Converts a folder of `.md` files into a single HTML dashboard; Static reports w/o JavaScript; Hostable anywhere.
Core Mechanism
  • Scans current directory for Markdown files; Reads each Markdown file; Converts Markdown content to HTML; Concatenates HTML into a template; Saves as `dashboard.html`.
Libraries Used
  • `markdown`, `os`.
Generation Process
  • Iterates `os.listdir('.')` for `.md` files; `markdown.markdown(f.read())` → converts Markdown to HTML; `TEMPLATE = '<html><body>{}</body></html>'` → basic HTML wrapper; `out += f'<h2>{file}</h2>{html}'` → appends each file's content w/ a heading; Writes final HTML to `dashboard.html`.
Key Learnings
  • Markdown parsing; HTML generation; Static site creation principles; File I/O.
Special Aspect
  • Local, fast, and owned reporting; Alternative to Notion/Google Docs for sharing progress; Zero JavaScript dependency.
6. Codebase Linguist
  • Scans project directories; Reports language distribution by file extension; Reveals actual coding language percentages.
Core Mechanism
  • Walks through directory tree; Extracts file extensions; Counts occurrences of each extension; Calculates percentage of total files for each extension.
Libraries Used
  • `os`, `collections.Counter`.
Analysis Process
  • `code_stats(root='.')` function; `os.walk(root)` → traverses directory; `os.path.splitext(f)[1]` → extracts file extension; `collections.Counter()` → efficiently counts extensions; Calculates `c/total:.1%` for percentage display.
Key Learnings
  • File system traversal; Data aggregation (`collections.Counter`); Basic statistical reporting.
Special Aspect
  • Deeply revealing about actual project composition; Shows often surprising language distribution (e.g., 40% Markdown, 30% config, 20% Python).
7. Context-Driven Clipboard Assistant
  • Clipboard watcher; Transforms copied data based on content; Provides programmable clipboard upgrade.
Core Mechanism
  • Continuously monitors clipboard content; Detects changes in clipboard data; Attempts transformations (JSON pretty print, URL extraction, uppercase); Copies transformed data back to clipboard.
Libraries Used
  • `pyperclip`, `json`, `re`, `time`.
Transformation Logic
  • `transform(data)` function; `json.dumps(json.loads(data), indent=2)` → attempts JSON pretty-printing; `re.findall(r'https?://\S+', data)` → extracts URLs; `data.upper()` → default transformation to uppercase if others fail.
Monitoring Loop
  • `while True:` loop; `pyperclip.paste()` → gets current clipboard content; `if data != prev:` → checks for new content; `pyperclip.copy(transform(data))` → writes transformed data back; `time.sleep(1)` → checks every second.
Key Learnings
  • Clipboard interaction; Regular expressions (`re`); JSON parsing; Real-time data processing.
Special Aspect
  • Invisible productivity boost; Live, programmable clipboard; Adapts clipboard content to context.
8. Terminal Time Machine
  • Snapshots and restores directory states; 'Undo' chaos in project folders; Git-like functionality w/o Git commits.
Core Mechanism
  • Creates timestamped ZIP archives of a specified folder; Extracts contents of a ZIP archive to restore a folder state.
Libraries Used
  • `os`, `shutil`, `datetime`, `zipfile`.
Snapshot Function
  • `snapshot(folder)` function; `name = f'backup_{datetime.date.today()}.zip'` → creates unique backup name; `zipfile.ZipFile(name, 'w')` → creates ZIP archive; `os.walk(folder)` → traverses folder; `z.write(os.path.join(root, f))` → adds each file to ZIP.
Restore Function
  • `restore(zip_file, target)` function; `zipfile.ZipFile(zip_file)` → opens ZIP archive; `z.extractall(target)` → extracts all contents to target folder.
Key Learnings
  • File archiving (`zipfile`); Directory management (`os`, `shutil`); Basic version control concepts.
Special Aspect
  • Git for lazy days; Ideal for quick prototyping; Undoes project folder changes easily.
