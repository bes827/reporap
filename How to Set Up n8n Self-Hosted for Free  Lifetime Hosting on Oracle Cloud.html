Step 1: Getting Your Server Ready
  • Initial setup on Oracle Cloud: sign up & instance creation.
  • Crucial for foundational server access & security.
Oracle Cloud Account & Instance
  • Sign up @ oracle.com/cloud/free/.
  • Create Ubuntu 22.04 instance (virtual server).
  • Record public IP & SSH key; essential for access.
Logging In via SSH
  • Use terminal/PowerShell: `ssh -i ~/your-key-path/ssh-key-2025-07-03.key ubuntu@<your-ip>`.
  • Replace `~/your-key-path/ssh-key-2025-07-03.key` w/ actual key path.
  • Replace `<your-ip>` w/ server's public IP (e.g., 80.225.122.46).
Securing SSH Key
  • Command: `chmod 400 ~/Downloads/n8n-demo-key/ssh-key-2025-07-03.key`.
  • Purpose: Locks down key permissions → only owner can read it.
  • Benefit: Prevents unauthorized access to server.
Updating Ubuntu & Installing Tools
  • Command: `sudo apt update && sudo apt upgrade -y && sudo apt install -y build-essential curl wget git ufw ca-certificates gnupg lsb-release software-properties-common`.
  • `build-essential`: for compiling software.
  • `curl`, `wget`: for downloading files.
  • `git`: for version control.
  • `ufw`: Uncomplicated Firewall for security.
  • `ca-certificates`: for secure web connections.
  • `gnupg`: for encryption.
  • `lsb-release`: identifies Linux distribution.
  • `software-properties-common`: manages software sources.
Step 2: Setting Up Node.js
  • n8n requires Node.js to run.
  • NVM (Node Version Manager) is used for flexible Node.js version management.
Installing NVM
  • Command: `curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash`.
  • Then: `export NVM_DIR="$HOME/.nvm"`.
  • Finally: `source "$NVM_DIR/nvm.sh"` → makes NVM available in current terminal session.
Installing Node.js v22
  • Command: `nvm install 22`.
  • Activate: `nvm use 22`.
  • Set as default: `nvm alias default 22`.
  • Verify: `node -v` → should show `v22.something`.
Step 3: Running n8n with PM2
  • PM2 ensures n8n runs continuously in the background.
  • Prevents n8n from stopping when terminal closes or server restarts.
Installing PM2
  • Command: `npm install -g pm2`.
  • Verify: `pm2 -v`.
Creating n8n Startup Script
  • Install `nano` editor: `sudo apt install -y nano`.
  • Create script: `nano ~/start-n8n.sh`.
  • Script content:
  • `#!/bin/sh`
  • `# Loads env variables from ~/.n8n/.env`
  • `set -a`
  • `. ~/.n8n/.env`
  • `set +a`
  • `# Starts n8n`
  • `npx n8n`
  • Make executable: `chmod +x ~/start-n8n.sh`.
  • Purpose: Loads environment variables & starts n8n.
Running n8n with PM2
  • Start n8n: `pm2 start ~/start-n8n.sh --name n8n`.
  • Enable auto-run on boot: `pm2 startup`.
  • Save PM2 configuration: `pm2 save`.
Key PM2 Commands
  • `pm2 start <app>`: Starts an application.
  • `pm2 stop <app>`: Pauses an application.
  • `pm2 restart <app>`: Restarts an application.
  • `pm2 status`: Shows status of all running applications.
  • `pm2 save`: Saves current process list for automatic restart on boot.
Step 4: Adding NGINX
  • NGINX acts as a reverse proxy.
  • Handles web traffic & forwards it to n8n running on port 5678.
  • Provides a cleaner access point for n8n.
Installing NGINX
  • Command: `sudo apt update && sudo apt install -y nginx`.
  • Enable NGINX to start on boot: `sudo systemctl enable nginx`.
  • Start NGINX service: `sudo systemctl start nginx`.
  • Verify status: `sudo systemctl status nginx`.
Configuring NGINX for n8n
  • Create config file: `sudo nano /etc/nginx/sites-available/n8n`.
  • Add server block configuration:
  • `server {`
  • ` listen 80;`
  • ` server_name your-domain.com; # Use your domain or IP`
  • ` location / {`
  • ` proxy_pass http://localhost:5678;`
  • ` proxy_http_version 1.1;`
  • ` proxy_set_header Upgrade $http_upgrade;`
  • ` proxy_set_header Connection "upgrade";`
  • ` proxy_set_header Host $host;`
  • ` proxy_set_header X-Real-IP $remote_addr;`
  • ` proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;`
  • ` proxy_set_header X-Forwarded-Proto $scheme;`
  • ` proxy_buffering off;`
  • ` proxy_cache off;`
  • ` chunked_transfer_encoding off;`
  • ` }`
  • `}`
  • Replace `your-domain.com` w/ actual domain or server IP.
  • Link config to `sites-enabled`: `sudo ln -s /etc/nginx/sites-available/n8n /etc/nginx/sites-enabled/`.
  • Test NGINX config: `sudo nginx -t`.
  • Reload NGINX: `sudo systemctl reload nginx`.
Step 5: Locking It Down with SSL
  • Secures communication w/ HTTPS.
  • Uses Certbot to obtain & manage free SSL certificates from Let's Encrypt.
Installing Certbot
  • Command: `sudo apt install -y certbot python3-certbot-nginx`.
Obtaining SSL Certificate
  • Command: `sudo certbot --nginx -d your-domain.com`.
  • Replace `your-domain.com` w/ your actual domain.
  • Follow prompts to complete setup.
Auto-Renewal of Certificate
  • Command: `sudo systemctl enable certbot.timer`.
  • Purpose: Automates SSL certificate renewals.
Configuring Server Firewall (iptables)
  • Edit iptables rules: `sudo nano /etc/iptables/rules.v4`.
  • Define rules to restrict access (e.g., allow specific IPs for SSH).
  • Apply/restore rules: `sudo iptables-restore < /etc/iptables/rules.v4`.
Step 6: Keeping Things Smooth
  • Ongoing maintenance & troubleshooting tips.
  • Ensures n8n remains secure, functional, & up-to-date.
Environment Variables (.env file)
  • Create/edit file: `~/.n8n/.env`.
  • Add variables:
  • `N8N_BASIC_AUTH_ACTIVE=true`
  • `N8N_BASIC_AUTH_USER=yourusername`
  • `N8N_BASIC_AUTH_PASSWORD=yourpassword`
  • `WEBHOOK_TUNNEL_URL=https://your-domain.com/`
  • Secure file: `chmod 600 ~/.n8n/.env` → restricts access to owner.
Checking Logs
  • Command: `pm2 logs n8n`.
  • Purpose: Troubleshoot n8n issues.
Firewall Configuration (UFW)
  • Allow SSH: `sudo ufw allow 22`.
  • Allow HTTP: `sudo ufw allow 80`.
  • Allow HTTPS: `sudo ufw allow 443`.
  • Enable UFW: `sudo ufw enable`.
Backups Strategy
  • n8n configs: Weekly backup to cloud storage.
  • .env file: Monthly backup to encrypted drive.
  • PM2 list: Monthly `pm2 save` + Git.
  • Scripts: After changes, commit to Git repo.
Updates
  • Regularly check for updates to n8n, Node.js, PM2, & NGINX.
  • Benefit: Prevents glitches & ensures security.

n8n Not Working
